03.

How I Build a Pipeline

A typical ELT flow I design and operate, from raw source to a query-ready warehouse. Select a stage to see the tools and what I optimize for.

Extract from source systems

Pull data from operational databases, ERPs like SAP, and third-party APIs. I prioritize log-based CDC over batch polling wherever the source supports it, to keep load on production systems minimal.

Debezium REST/SOAP APIs SAP
Optimizing for: source-system safety

Land raw data, unaltered

Raw events and snapshots land in a staging zone exactly as received. Keeping an immutable raw layer means I can always replay or reprocess history if a downstream model changes.

Redpanda / Kafka Cloud Storage Snowflake Stages
Optimizing for: replayability

Model into trusted layers

Raw data moves through staging, intermediate, and mart layers with version-controlled SQL. Every model has tests for nulls, uniqueness, and referential integrity before it's allowed downstream.

DBT Airflow PySpark
Optimizing for: data quality

Serve query-ready marts

Final marts are documented, access-controlled, and exposed to BI tools and stakeholders. I track freshness SLAs so business users always know how current a number is.

Power BI Looker Snowflake
Optimizing for: trust & freshness
